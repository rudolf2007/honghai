網絡框IPv6

 

互聯網正在從IPv4遷移到IPv6。IPv4地址空間已經耗盡，新的服務將不得不使用IPv6進行部署。作為這種遷移的一部分，大多數UTM供應商都提供了一個雙棧解決方案，支持同一設備上的IPv4和IPv6網絡堆棧，在這方面，Network Box並沒有什麼不同。NBRS-5.0包括IPv4和IPv6網絡堆棧，並且對兩種協議（對於目前（IPv4）以及未來（IPv6）設計的網絡架構）都有很好的支持。

 

但是，NBRS-5.0與其他方面的區別在於我們超越了僅僅支持IPv6，而是提供產品功能和服務來幫助您遷移到IPv6。我們將雙棧與協議轉換相結合。

 

使用路由器級技術，結合高級代理和服務，NBRS-5.0支持IPv4和IPv6之間的雙向轉換，允許IPv4客戶端連接到IPv6服務器，反之亦然。

 

網絡盒NBRS-5.0 IPv6

 

專家們一直在預測今天互聯網協議IPv4即將死亡二十多年，但不知何故，它一直在繼續下去。它最後的“緩解”大量部署源NAT技術和RFC 1918私有地址（10/8,172.16 / 12和192.168 / 16）。但是，隨著去年將區域互聯網註冊管理機構（RIR）的最終五/八個網絡塊分發到2011年1月31日，IANA的用盡以及2011年4月15日的RIR APNIC用盡，終於看起來太陽正在設置IPv4 。剩下的雷達預計會在幾年內耗盡游泳池。

 

什麼是大事？

 

那麼“什麼大事呢？”，我們聽到有人說 - “所有現代設備都有IPv6支持”。

 

那麼最緊迫的問題是缺乏經驗和測試。IPv4擁有數十年的現實世界經驗。有數十億個設備談論協議，大多數互操作性問題早已解決，任何不兼容性都比較容易看到。IPv6並非如此。

 

現在，今天，我們擁有大量的採用雙棧方式的設備，與IPv6堆棧一起運行其IPv4網絡堆棧。不同之處在於IPv6堆棧幾乎沒有進行任何測試。IPv6協議本身也比IPv4的前身要復雜得多。因此，儘管您的互聯網連接的烤麵包機可能同時支持IPv4和IPv6，但與IPv6相比，您可以更加確信IPv4的工作。

添加兩個堆棧之間的互操作性問題（包括必須對特定網絡連接使用哪個堆棧的決定），您可以開始看到問題。

 

大爆炸事件

 

如果IPv4明天被關閉，並且每個人都切換到IPv6，混亂就會統治著。這兩個協議是不兼容的，絕大多數互聯網用戶甚至沒有來自ISP的IPv6連接（大量的ISP甚至不支持IPv6在其網絡上）。

 

通過首先提供IPv6作為選擇，然後使用自動雙棧交換機制，然後最終喜歡IPv6和逐步淘汰IPv4，更明智的做法是逐漸過渡到IPv6。這是今天使用的方法。

 

世界IPv6日是2011年6月8日。這是發布AAAA（IPv6）和A（IPv4）地址記錄以及連接問題評估的效果的24小時測試。該測試在參與的大型400家公司中取得了很大成功，證明了互聯網上最常見的網站（谷歌，Facebook等）仍然可以在自動雙IPv4-IPv6世界中訪問。

 

世界IPv6發布日

 

雖然Network Box NBRS-3.0像我們大多數競爭對手一樣都對IPv6進行了基本的支持，但使用類似於我們的競爭對手的雙棧方式，它只對協議的更高級代理支持有限。

 

NBRS-5.0進一步，為各級IPv4和IPv6協議提供全面的本地支持。此外，NBRS-5.0支持翻譯服務，以協助這兩種協議之間的共存和遷移。使用路由器級技術，結合高級代理和服務，NBRS-5.0將支持IPv4和IPv6之間的雙向轉換。這意味著具有內部僅IPv4環境的客戶可以使用其NBRS-5.0網絡盒接受IPv4和IPv6流量，並將其轉換為IPv4基礎設施。

 

對於沒有來自其ISP的IPv6連接的客戶，NBRS-5.0包括支持IPv6-in-IPv4隧道，以通過現有的IPv4 ISP鏈路提供IPv6連接。

 

支持IPv6的目標指導了自開始開始以來NBRS-5.0的許多設計和實施決策，特別是系統級配置中間件組件的設計和實現。從NBRS-5.0的角度來看，IPv6是一流的公民，與傳統的IPv4一樣，得到認可和支持。

 

NBRS-5.0 IPv6就緒

 

除了在中間件層支持IPv6的成功之外，Network Box已經意識到需要有意義地演示IPv6網絡層的IPv6支持的正確性和合規性。Network Box承諾說，當NBRS-5.0發佈時，將通過ipv6ready.org認證到目前的第2階段。這是行業最全面和認可的IPv6準備認證。

 

IPv6 Ready是國際公認的認證，經過全面的測試，可以測量產品IPv6網絡堆棧與官方IPv6互聯網標準中定義的格式，功能和行為的兼容性，授予供應商產品。這是確保我們今天的互操作性最接近的事情。

 

Network Box花了大量時間和精力，精細調整NBRS-5.0 IPv6網絡層的實施和配置，最近成功通過了IPv6初步評估，這是對官方測試執行的評估的直接反映實驗室。

 

Network Box即將提交NBRS-5.0，用於核心協議測試規範的IPv6 Ready-Phase 2標準認證。

 

鑑於NBRS-5.0在初步評估中取得的成功，全面的IPv6就緒認證只是一步之遙，這將使Network Box成為一個由思科，瞻博網絡，IBM，富士通等名稱填充的供應商。此外，Network Box NBRS-5.0將成為首個實現IPv6 Ready認證的香港開發產品，也可能是世界上第一個以最新版本的測試規範為最終的產品。的2011年。

  

結論

 

NBRS-5.0旨在幫助我們的客戶遷移到IPv6，而不僅僅是在網絡上充當IPv6設備。在啟動時，將通過IPv6 Ready認證獲得核心協議測試規範的第2階段標準。

 

你知道嗎？

Network Box是第一個，迄今為止，只有託管安全服務提供商才能實現IPv6 Ready核心階段2認證。
